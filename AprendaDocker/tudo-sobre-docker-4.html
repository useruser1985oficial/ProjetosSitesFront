    Importando e Exportando Imagens, Containers e Volumes

    Para exportar uma imagem em um arquivo tar, digite isso no prompt:

    
docker save -o minhaimagem.tar meuapp:latest

    Se quiser comprimir:

    
docker save meuapp:latest | gzip > minhaimagem.tar.gz

    E para importar no outro computador, faça assim:

    
docker load -i minhaimagem.tar

    Para exportar um container, digite esses comandos na ordem:

    
docker commit meucontainer meuappcustom:latest

docker save -o meuappcustom.tar meuappcustom:latest

    E para restaurar no outro computador:

    
docker load -i meuappcustom.tar

# Depois rode o docker run normalmente com todos os parâmetros.

    Para fazer o backup de volumes, faça assim:

    
# Não esqueça do ponto:
docker run --rm \
  -v dadosvolume:/volume \
  -v $(pwd):/backup \
  alpine \
  tar czf /backup/dadosvolume.tar.gz -C /volume .

    Isso gerará um tar com o volume.

    Para restaurar no outro computador:

    
docker volume create dadosvolume

docker run --rm \
  -v dadosvolume:/volume \
  -v $(pwd):/backup \
  alpine \
  sh -c "cd /volume && tar xzf /backup/dadosvolume.tar.gz"

    Para migrar uma aplicação completa, normalmente fazemos assim:

    • docker-compose.yml
    • imagem.tar
    • volume1.tar.gz
    • volume2.tar.gz

    E no computador de destino:

    
docker load -i imagens.tar

docker volume create volume1

docker volume create volume2

docker compose up -d

    Trabalhando com Imagens no Docker Hub

    Para enviar uma imagem para o Docker Hub, fazemos uma tag da imagem com o nome do seu repositório e depois faz push.

    Primeiro acesse o Docker Hub em https://hub.docker.com/, e depois no prompt, faça o login com docker login e digite seu usuário e senha (ou access token), depois digite docker images para verificar a imagem local. Para criar uma tag, faça assim:

    
docker tag meuapp:latest nomedeusuario/meuapp:latest

    No Docker Hub, crie um repositório com o nome do seu app (deve ser o mesmo que a imagem), e envie a imagem com esse comando:

    
docker push nomedeusuario/meuapp:latest

    Para baixar a mesma imagem em outro computador, faça o login também e digite esse comando:

    
docker pull nomedeusuario/meuapp:latest

    E execute normalmente:

    
docker run -d nomedeusuario/meuapp:latest

    PS: Se o repositório for privado, apenas usuários autorizados podem fazer pull. E volumes e containers não são armazenados no Docker Hub, apenas imagens. Docker Compose também não são armazenados no Docker Hub, mas podemos versionar ele num repositório baseado em Git.
