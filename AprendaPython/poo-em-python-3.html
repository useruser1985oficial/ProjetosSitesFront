    Programção Orientada a Objetos em Python - Parte 3

    Encapsulamento: Getters, Setters e Property

    Como explicado, o encapsulamento visa manter a integridade do sistema, protegendo o estado interno do objeto contra interferência externa não regulamentada. No caso do Python nós apenas sinalizamos quais atributos devem ser considerados protegidos ou privados. Para podermos manipular os atributos protegidos e privados precisaremos criar métodos públicos conhecidos como getters e setters, que controlarão o acesso de forma mais segura.

    Voltando ao exemplo da conta bancária, nós não teremos acesso direto ao cofre pra colocar e pegar dinheiro, para isso teremos um funcionário do banco para poder fazer isso de forma controlada, esse é o conceito de métodos acessores.

    No Python existem duas maneiras de permitir o acesso aos dados encapsulados: O uso de getters e setters, e o uso do decorador @property.

    O exemplo de getters e setters poderia ser, por exemplo, na nota de um aluno, cuja nota seria protegida, para apenas pegar o valor da nota, usamos um getter, da mesma forma, para mudar, usamos um setter.

    Vamos supor esse código:

    
from rich import print, inspect

class Avaliacao:
    def __init__(self, nome, disciplina, nota = 0):
        self.nome = nome
        self.disciplina = disciplina
        self.nota = nota # Por enquanto deixe público

    Código principal:

    
def main():
    av1 = Avaliacao("Sérgio", "Matemática", 9.5)
    inspect(av1)

if __name__ == "__main__":
    main()

    Só que isso poderá dar erros, como por exemplo:

    
av1 = Avaliacao("Sérgio", "Matemática", 9.5)

av1.nota = -234 # Não deveria permitir

inspect(av1)

    Aí, na classe, colocamos o atributo nota como protegido, assim:

    
self._nota = nota

    E exiba assim:

    
av1 = Avaliacao("Sérgio", "Matemática", 9.5)

av1.nota = -234

inspect(av1, private = True)

    Nesse caso acima, ele protegeu o atributo nota da classe, e deixou a nota exposta no código principal isolada.

    Para criar os métodos getter (que pega) e setter (que muda), alteramos a classe assim:

    
class Avaliacao:
    def __init__(self, nome, disciplina, nota = 0):
        self.nome = nome
        self.disciplina = disciplina
        self._nota = nota
    
    # Métodos acessores
    def getNota(self): # Método público getter, pega a nota
        return self._nota
    
    def setNota(self, nota): # Método público setter, muda a nota
        self._nota = nota

    PS: Esses nomes são os mais usados, mas eles podem ter qualquer nome.

    Só que tem um porém, através dos métodos setters poderemos mudar o atributo privado diretamente:

    
av1 = Avaliacao("Sérgio", "Matemática", 9.5)

av1.setNota(-234)

inspect(av1, private = True)

    Então, nesse caso, podemos personalizar o setter, dessa forma:

    
def setNota(self, nota): # Método público setter, muda a nota
    if nota >= 0 and nota <= 10:
        self._nota = nota
    else:
        print("Nota Inválida!")

    Se ele passar um valor fora do intervalo, ele não alterará:

    
av1 = Avaliacao("Sérgio", "Matemática", 9.5)

av1.setNota(10) # Passe valores válidos e inválidos

inspect(av1, private = True)

    Para usar o getter, basta usar ele num print ou atribuir o valor de retorno a outra variável:

    
av1 = Avaliacao("Sérgio", "Matemática", 9.5)

av1.setNota(10)
print(av1.getNota())

print(f"{av1.nome} tirou {av1.getNota()} em {av1.disciplina}.")

inspect(av1, private = True)

    Podemos usar o decorador @property em Python e outras linguagens modernas para acessar atributos protegidos e privados. Nesse caso usamos atributos especiais com esse indicador para validar o atributo. Vamos alterar o código da classe assim:

    
class Avaliacao:
    def __init__(self, nome, disciplina, nota = 0):
        self.nome = nome
        self.disciplina = disciplina
        self._nota = nota

    # Criando Atributos Validáveis:
    @property
    def nota(self): # Esse método funcionará como um atributo que pega o valor
        return self._nota

    @nota.setter # Nome do property, mesmo nome do atributo sem underline
    def nota(self, no): # Esse método funcionará como um atributo que muda o valor
        if no >= 0 and no <= 10:
            self._nota = no
        else:
            print("Nota Inválida!")

    E no código principal:

    
av1 = Avaliacao("Sérgio", "Matemática")

av1.nota = 3.5 # Como um setter, tente passar valores válidos e inválidos
print(av1.nota) # Como um getter

print(f"{av1.nome} tirou {av1.nota} em {av1.disciplina}.") # Como um getter

inspect(av1, private = True)

    PS: Além de setter, podemos ver o decorador getter, como por exemplo, no caso acima, seria @nota.getter, e temos também o deleter, como o @nota.deleter, dessa forma:

    
@nota.deleter
def nota(self):
    pass
